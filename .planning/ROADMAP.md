# Roadmap

## Delivery Strategy

- Mode: ultra-lean milestones (3 to 5 days)
- Target: fast MVP-to-beta rollout
- Canonical planning: this repo
- Code implementation: separate app repo

## Milestones

| Milestone | Window | Primary Outcome | Exit Gate |
|---|---|---|---|
| M0 | Day 0-2 | Architecture + docs bootstrap | Decision-complete docs + task board |
| M1 | Day 3-7 | UPI ingest accuracy core | >=95% amount extraction |
| M2 | Day 8-12 | Categorization v1 | >=90% category precision |
| M3 | Day 13-17 | Summaries + visualization | Daily/weekly/monthly validated |
| M4 | Day 18-22 | Insights + budgeting | Explainable insights + alert thresholds |
| M5 | Day 23-27 | Auth hardening + beta prep | Stable OTP flows + release candidate |
| M6 | Day 28-30 | Quality gate + beta decision | P0/P1 cleared + go/no-go |

## Tracking Rules

- Every session updates `TASK_BOARD.md` and `STATUS_LOG.md`.
- Every decision change updates `DECISIONS.md`.
- Every new blocker updates `RISKS.md`.
