# Phases

## Phase 0: Planning and Foundations

### Goals
- Lock architecture and execution rules.
- Create full planning artifact set.
- Prepare first implementation-ready tasks.

### Tasks
- P0-T1 Create canonical planning docs.
- P0-T2 Define branch/commit/PR workflow.
- P0-T3 Convert requirements into executable task IDs.

### Exit Criteria
- Docs created and linked.
- First 2 implementation sessions pre-scoped.

## Phase 1: Ingest Accuracy (Top 4 UPI Apps)

### Goals
- High-confidence extraction for GPay, PhonePe, Paytm, BHIM.
- Low-confidence routing to review queue.

### Exit Criteria
- >=95% amount extraction on validation corpus.
- Deduplication verified.

## Phase 2: Categorization Engine v1

### Goals
- 8-10 category ontology.
- Rule-based categorization + user correction learning.

### Exit Criteria
- >=90% precision on labeled set.
- Single-tap recategorization live.

## Phase 3: Summaries and Visualizations

### Goals
- Daily/weekly/monthly summaries.
- Donut, bar, line visualizations.

### Exit Criteria
- Aggregation reconciliation complete.
- UX readable and layered.

## Phase 4: Insights and Budgeting

### Goals
- Explainable weekly insights.
- Budget thresholds with in-app + push alerts.

### Exit Criteria
- Suggestions traceable to metrics.
- 80% and 100% alert logic verified.

## Phase 5: Auth and Beta Readiness

### Goals
- Firebase phone OTP reliability.
- Closed beta release readiness.

### Exit Criteria
- Stable auth funnel metrics.
- RC build delivered for testers.

## Phase 6: Quality Gate and Rollout Decision

### Goals
- End-to-end quality validation.
- Go/no-go decision.

### Exit Criteria
- KPI gate passed.
- P0/P1 issues resolved.
